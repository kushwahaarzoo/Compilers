# Lex-Yacc Based plain English Text conversion to SQL Queries

## 1. The Language of the Grammar Written

The grammar is written using **Lex (Flex)** and **Yacc (Bison)**, which are standard tools used in compiler design.

* **Lex (.l file)** is used for **lexical analysis**. It defines patterns using regular expressions to identify tokens from the input. These tokens represent keywords, identifiers, numbers, and operators in the restricted English input.

* **Yacc (.y file)** is used for **syntax analysis**. It defines grammar rules using context-free grammar (CFG) to parse the sequence of tokens generated by Lex.

* The overall language supported by this grammar is a **restricted form of English**, designed specifically to represent database queries. 

---

## 2. The Logic Behind the Rules Specified in Each Program

### A. Lex Program (.l file)

The Lex program is responsible for converting input text into tokens.

1. **Ignoring Irrelevant Words**
   Common filler words such as "the", "a", and "me" are ignored because they do not contribute to the meaning of the query.

2. **Handling Synonyms**
   Different words with similar meanings are mapped to a single token.
   For example, words like "give", "show", and "display" are all mapped to a common token such as `GET`. This simplifies parsing and allows flexible input.

3. **Keyword Recognition**
   Specific words related to the database domain are mapped to tokens, such as:

   * "students" → STUDENTS
   * "name" → NAME
   * "cpi" → CPI

4. **Condition Handling**
   Words representing conditions are converted into relational operator tokens:

   * "greater" → GT
   * "less" → LT
   * "where" → WHERE

5. **Handling Numbers and Identifiers**

   * Numeric values are captured and stored using `yylval` for use in parsing.
   * General words (like names) are treated as identifiers.

6. **Sentence Termination**
   Punctuation symbols such as ".", "?", or ";" are used to mark the end of a statement and return an `END` token.

---

### B. Yacc Program (.y file)

The Yacc program defines grammar rules and performs syntax-directed translation to SQL.

1. **Program Structure**
   The grammar allows one or more statements. Each statement is parsed independently.

2. **Statement Types**
   The grammar supports different types of database operations:

   * SELECT queries
   * INSERT queries
   * UPDATE queries
   * DELETE queries

3. **SELECT Query Rules**

   * Rules are defined to handle requests like retrieving all records or specific columns.
   * Example logic: "get all students" is translated into `SELECT * FROM Student`.
   * Column-specific queries generate SQL using the parsed column list.

4. **Conditional Queries (WHERE Clause)**

   * Conditions such as "CPI greater than 8" are parsed and translated into SQL conditions like `CPI > 8`.
   * These are incorporated into the WHERE clause of SQL statements.

5. **Insert Statement Logic**

   * English patterns like "insert new student with values" are mapped to SQL `INSERT INTO` statements.
   * Values are extracted and placed appropriately in the query.

6. **Update Statement Logic**

   * Statements like "update student set name to X" are converted into SQL `UPDATE` queries.
   * The grammar identifies fields and new values and constructs the SET clause.

7. **Delete Statement Logic**

   * Patterns such as "delete students where CPI less than 8" are translated into SQL `DELETE` queries with conditions.

8. **Syntax Directed Translation (SDT)**

   * Each grammar rule contains semantic actions written in C.
   * These actions generate SQL queries directly during parsing.
   * Symbols like `$1`, `$2` are used to access components of parsed rules.

---
## 3. Grammar Used

The grammar defined in the Yacc file is a **Context-Free Grammar (CFG)** that parses restricted English queries and maps them to SQL operations.

### Start Symbol

```
program
```

### Grammar Rules

```
program
    : program stmt
    | stmt
    ;

stmt
    : GET request END
    | PLEASE GET request END
    | insert_stmt END
    | update_stmt END
    | delete_stmt END
    ;

request
    : ALL STUDENTS
    | column_list STUDENTS
    | column_list STUDENTS WHERE condition
    ;

column_list
    : column
    | column_list AND column
    ;

column
    : NAME
    | CPI
    | ID
    ;

condition
    : CPI GT NUM
    | CPI LT NUM
    ;

insert_stmt
    : INSERT NEW_STUDENT VALUES value_list
    ;

value_list
    : value
    | value_list ',' value
    ;

value
    : NUM
    | ID
    ;

update_stmt
    : UPDATE STUDENTS SET column '=' value
    ;

delete_stmt
    : DELETE STUDENTS WHERE condition
    ;
```

### Explanation

* The grammar supports multiple statements using recursion in `program`.
* `stmt` defines different types of valid queries.
* `request` handles SELECT queries with optional conditions.
* `column_list` allows multiple attributes using conjunction (AND).
* `condition` defines comparison operations.
* Separate rules exist for INSERT, UPDATE, and DELETE operations.

## 4. Syntax Directed Translation (SDT)

The grammar is augmented with **semantic actions (C code)** to directly generate SQL queries during parsing.

### Example SDT Rules

#### SELECT All Records

```
request
    : ALL STUDENTS
      { printf("SELECT * FROM Student"); }
```

#### SELECT Specific Columns

```
request
    : column_list STUDENTS
      { printf("SELECT %s FROM Student", $1); }
```

#### SELECT with Condition

```
request
    : column_list STUDENTS WHERE condition
      { printf("SELECT %s FROM Student WHERE %s", $1, $4); }
```

#### Column List Handling

```
column_list
    : column
      { $$ = $1; }
    | column_list AND column
      { $$ = strcat($1, ", "), strcat($$, $3); }
```

#### Condition Handling

```
condition
    : CPI GT NUM
      { sprintf($$, "CPI > %d", $3); }
    | CPI LT NUM
      { sprintf($$, "CPI < %d", $3); }
```

#### INSERT Statement

```
insert_stmt
    : INSERT NEW_STUDENT VALUES value_list
      { printf("INSERT INTO Student VALUES (%s)", $4); }
```

#### UPDATE Statement

```
update_stmt
    : UPDATE STUDENTS SET column '=' value
      { printf("UPDATE Student SET %s = %s", $4, $6); }
```

#### DELETE Statement

```
delete_stmt
    : DELETE STUDENTS WHERE condition
      { printf("DELETE FROM Student WHERE %s", $4); }
```

---

## Explanation of SDT

* Each production rule is associated with a **semantic action**.
* Symbols like `$1`, `$2`, etc., refer to components of the rule.
* `$$` represents the result of the current rule.
* Functions like `printf`, `sprintf`, and `strcat` are used to construct SQL queries.
* Translation is performed **during parsing**, so no separate intermediate structure (like AST) is required.
